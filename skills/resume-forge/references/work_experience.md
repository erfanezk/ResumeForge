# Work Experience Deep Dive

The work experience section is the most important part of your resume. Recruiters spend the most time here — if your bullets are weak, everything else fails.

## The HR Test

HR reads bullets before any technical person. If a non-technical reader can't understand the purpose of what you did, rewrite it.

**Structure:** Why (satisfies HR) + How (satisfies technical reviewer)

- Bad: "Used caching and indexing" — Why? HR has no idea.
- Good: "Reduced API response time by 60% by implementing Redis caching and adding database indexes, cutting page load time from 3s to 1.2s for 50K daily users"

## Bullet Writing by Role Level

### Entry-Level Developer (0-2 years)

Focus on: learning, contributions, tools learned, small project results.

| Component | Template | Example |
|-----------|----------|---------|
| Build/learn | `[BUILT/LEARNED] [WHAT] using [TOOLS], [OUTCOME]` | "Built a data validation service using Python and PostgreSQL, replacing a manual spreadsheet process for the ops team" |
| Skill applied | `[APPLIED] [SKILL] to [TASK], [RESULT]` | "Applied Docker and Kubernetes to containerize the ML inference pipeline, reducing deployment time from 2 hours to 8 minutes" |
| Collaboration | `[COLLABORATED WITH] [TEAM] to [OUTCOME]` | "Collaborated with 2 senior engineers to fix a memory leak, reducing API response time by 30%" |
| Contribution | `[CONTRIBUTED TO] [WHAT], [IMPACT]` | "Contributed to an open-source monitoring library, now used by 3 internal teams" |

**Example:**
```
Software Engineering Intern - TechStart Inc | San Francisco, CA | 06/2023 - 08/2023
- Built a data validation service using Python and PostgreSQL, replacing
  a manual spreadsheet process for the ops team
- Learned Kubernetes and Docker, then containerized the company's ML
  inference pipeline for faster deployment cycles
- Collaborated with 2 senior engineers to fix a memory leak, reducing
  API response time by 30%
- Contributed to an open-source monitoring library, now used by 3
  internal teams
```

## Bullet Count Guidelines by Experience Level

| Role Level | Bullets per Job | Total Recommended |
|-----------|----------------|-------------------|
| Entry-level (0-2 years) | 3-5 bullets | 6-10 total |
| Mid-level (2-5 years) | 3-5 bullets | 9-15 total |
| Senior (5-10 years) | 3-4 bullets | 9-12 total |
| Lead / Staff (10+ years) | 2-4 bullets | 6-10 total |

**Why not more bullets?** Recruiters spend 6 seconds on first scan. If they see 10 bullets per job, they read nothing. Quality over quantity — 3 great bullets beat 6 mediocre ones.

**How to cut:** Combine related points. "Reviewed PRs", "Mentored junior devs", "Set up CI" → "Established engineering standards through code reviews, team mentorship, and CI/CD setup"

**What goes in each bullet:** Each bullet should contain at least one of: What you did, How you did it (approach or tool), Result you achieved (measurable outcome). Result is optional but preferred. If you can't quantify, describe scope or impact qualitatively.

## Include vs. Exclude by Career Stage

| Should Include | Should Exclude |
|---------------|----------------|
| Measurable achievements (X% improvement, $Y revenue) | Day-to-day task descriptions |
| Scope (team size, system scale, user base) | Tasks you did once but don't own |
| Tools and technologies you chose/introduced | Things the whole team did |
| Cross-functional collaboration outcomes | Responsibilities without results |
| Technical decisions you made and why | Old/unrelated job details |

## Common Work Experience Flags

| Flag | Red Flag Example | Fixed Example |
|------|-----------------|---------------|
| No metrics | "Responsible for backend development" | "Led backend development for a platform serving 50K daily users" |
| Too many bullets (>7) | 8+ bullets for a single job | Combine related bullets, keep 3-5 strongest |
| Vague verbs | "Worked on", "Helped with", "Did" | "Built", "Designed", "Reduced", "Led" |
| Responsibility list | "Duties included X, Y, Z" | "Achieved X by doing Y" |
| Same bullet structure repeated | 4 bullets starting with "Developed..." | Vary action verbs and structure |
| Unverifiable claims | "Significantly improved performance" | "Reduced API p95 latency from 1.2s to 180ms" |
| Irrelevant content | "Worked as a barista while in college" (for senior dev) | Leave off or minimize to one line if very relevant |

## Rules

- ✅ 3-5 bullets per job — quality over quantity
- ✅ Reverse chronological order, most recent first
- ✅ Past tense, third person — "Built" not "I built"
- ✅ Strong action verbs: built, architected, led, optimized, migrated, automated
- ✅ Quantify results: X% improvement, $Y revenue, N users/scale
- ✅ Address employment gaps or be prepared to explain them
- ❌ Don't list every job — only relevant experience (last 10-15 years)
- ❌ Don't write paragraphs — bullet points only
- ❌ Don't use weak verbs: did, helped, worked on, assisted
- ❌ Don't repeat the same bullet structure for every entry
- ❌ Don't include unverifiable metrics — every number must be defensible in an interview
- ❌ Don't overlap dates between jobs (work + education overlap is OK)
