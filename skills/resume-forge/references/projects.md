# Projects Deep Dive

Projects showcase skills that work experience alone can't cover. They prove you can build — not just maintain.

## Why They Matter

- Demonstrate initiative beyond your day job
- Prove you can build complete solutions end-to-end
- Show depth in specific technologies
- Make abstract skills concrete and verifiable

**Rule:** Every project listed must be something you can discuss in an interview. If you can't explain the technical decisions, leave it off.

## Where to List

| Context | Placement | Example |
|---------|-----------|---------|
| Part of your job | Work Experience section as bullets | Under relevant role |
| Personal/open source | Dedicated "Projects" section | After Experience or Skills |
| Academic | Education section or Projects section | Capstone or thesis projects |
| Notable portfolio piece | Projects section + linked in summary | Custom URL to live project |
| Contributing to open source | Projects section or Skills section | GitHub link + contribution summary |

## What to Include Per Project

| Field | Description | Example |
|-------|-------------|---------|
| Project Name | Specific, descriptive name | "Real-Time Collaboration Engine" |
| Role | Your role in the project | "Solo developer" or "Lead developer (3-person team)" |
| Purpose/Goal | What problem it solves | "Enables real-time document editing for distributed teams" |
| Technologies | Key tools, languages, frameworks | "Python, WebSockets, Redis, PostgreSQL" |
| Results/Achievements | Quantifiable impact if available | "Handles 10K concurrent connections with <50ms latency" |
| Link | URL to repo, demo, or writeup | "github.com/johnsmith/collab-engine" |

## How to Format

### Template

```
[Project Name]
[Role] | [Technologies Used]
[Goal/purpose] — [results/achievements]
[Link if applicable]
```

### Example Entries

```
Personal Projects

Real-Time Collaboration Engine
Solo developer | Python, WebSockets, Redis, PostgreSQL
Built a real-time document editing system using operational
transformation, supporting 10K concurrent connections with <50ms
latency.
github.com/johnsmith/collab-engine

Open Source Contributions
Contributor | Python, GitHub Actions, Docker
Contributed 12 pull requests to FastAPI — including a feature
for async background tasks and documentation improvements.
Total: 1.2K GitHub stars on personal libraries.
```

## Which Projects to List

| Include | Exclude |
|---------|--------|
| Projects directly relevant to the target role | Outdated projects (last touched 5+ years ago) |
| Projects you can explain technically | Projects where you only did minor tasks |
| Projects with measurable results | Incomplete or abandoned projects (unless impressive) |
| Open source contributions with impact | Hobby projects unrelated to your career |
| Projects showing progression (from simple to complex) | Everything you've ever touched |

**Rule:** 3 impressive projects beat 10 mediocre ones. Quality over quantity.

## Before/After Transformations

| Before (Weak) | After (Strong) | Why |
|--------------|----------------|-----|
| "Worked on an open-source project" | "Contributed 12 pull requests to FastAPI, including an async task feature adopted by 2K+ developers" | Specific, quantified, verifiable |
| "Built a website for practice" | "Built a portfolio site with React and Tailwind, deployed to Vercel with 99.9% uptime" | Specific stack, real deployment |
| "Made some improvements to a project" | "Optimized the database layer using connection pooling, reducing query latency by 60%" | Specific improvement, measurable result |
| "Did a cool project" | "Built a real-time chat system using WebSockets and Redis, handling 5K concurrent users with <100ms message delivery" | Specific, technical, scalable |

## Role-Based Project Selection

| Role Type | What to Highlight |
|-----------|-----------------|
| Backend developer | API design, database architecture, performance optimization |
| Frontend developer | UI components, responsive design, state management |
| Full-stack | End-to-end feature delivery, team collaboration |
| DevOps | CI/CD pipelines, infrastructure automation, monitoring |
| Data engineer | Pipeline architecture, data modeling, ETL processes |

## Project Count Guidelines

| Experience Level | Recommended Projects |
|-----------------|---------------------|
| Entry-level (0-2 years) | 2-4 (can include academic and personal projects) |
| Mid-level (2-5 years) | 2-3 (work-related or significant personal projects) |
| Senior (5-10 years) | 1-2 (only if highly impressive or directly relevant) |
| Staff / Principal | 0-1 (work experience should speak for itself) |

## Rules

- ✅ List 2-5 most impressive, relevant projects
- ✅ Include purpose, technologies, role, and results for each
- ✅ Use the same bullet-point rules as work experience
- ✅ Quantify results when possible (10K users, 99.9% uptime, 2K stars)
- ✅ Include links (GitHub, live demo) when available
- ✅ Focus on what you built and what it does — not just what you used
- ❌ Don't include outdated or irrelevant projects
- ❌ Don't include incomplete projects unless they're actively being developed
- ❌ Don't list projects where you can only describe the tools, not the outcomes
- ❌ Don't pad with trivial projects to fill space
