# Skills Section Best Practices

## The Golden Rule

Only list skills you can talk about for 15 minutes in an interview. If you installed PostgreSQL once and know the basic SQL syntax, that's not a PostgreSQL skill. You should be able to discuss materialized views, window functions, recursive CTEs, special data types, and indexes.

## What to Include

- **Hard skills only** — programming languages, frameworks, tools, databases, cloud platforms, methodologies
- **Relevant to the target job** — tailor for each application
- **Supported by experience** — every skill should have at least one bullet point in your work experience that demonstrates it

## What to Exclude

- **Soft skills** — Leadership, Teamwork, Communication. Show these through your work experience entries instead.
- **Irrelevant skills** — Microsoft Word for a developer role, cooking for any professional role
- **Skills you just touched** — "I installed it once" doesn't count
- **Outdated skills** — technologies you haven't used in years

## Formatting Rules

- Just list the skills, no descriptions or ratings
- No star ratings or progress bars — these are subjective and not ATS-friendly
- No "Familiar with X" or "Expert in X" — just write the skill name
- Exception: if you have a valid certification, include it. "AWS (Certified Solutions Architect)" is fine.
- A valid certification means you passed an exam that's widely accepted as proof of competency

## Examples

**Bad (Python backend role):**
"Expert in Java, C++, Python, Machine Learning, MariaDB, PostgreSQL, MySQL, FastAPI, Django, SQL, Spring Boot, Docker"

This lists 12 skills. Most are unrelated to a Python backend role (Java, Spring Boot, C++), and many won't be backed by experience. The interviewer will grill you on all of them.

**Good (Python backend role):**
"Python, Docker, Django, FastAPI, SQL, PostgreSQL"

Six focused skills, all relevant, all presumably backed by work experience bullets.

## Certification Format

If you have certifications, format them clearly:
- AWS (Certified Solutions Architect) — earned via exam, widely recognized
- Scrum Master (Associate, Certified Scrum Master)
- Six Sigma Green Belt

Don't list certifications you haven't completed or that are expired.

## How to Tailor Skills to a Job Description

When applying to a specific role, follow this process:

1. **Copy** the required skills from the job description
2. **Filter** — only keep skills you can actually use in an interview
3. **Add proof** — for each skill, make sure there's a bullet in your experience that demonstrates it
4. **Organize** — group by category (languages, frameworks, tools, cloud)
5. **Limit** — keep to 5-8 skills. If you list 20, it looks like you don't actually know any of them deeply.

## Skill Count Guidelines

| Experience Level | Recommended Skill Count |
|-----------------|------------------------|
| Entry-level (0-2 years) | 5-8 skills |
| Mid-level (2-5 years) | 6-10 skills |
| Senior (5-10 years) | 5-8 skills (depth over breadth) |
| Staff / Principal | 4-6 skills (focus on strategic/leadership) |

**Rule:** Every skill should have at least one bullet in your experience that proves it. If you list "Kubernetes" but your bullets don't mention it, either add a bullet or remove it.
