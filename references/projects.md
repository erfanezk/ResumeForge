# Projects Section Deep Dive

## Be Selective

List only the most relevant projects that showcase skills directly related to the target job. Don't clutter with outdated or irrelevant projects.

## Provide Clear Descriptions

For each project include:
- **Project Name**: Specific name or title
- **Role**: Developer, manager, analyst, etc.
- **Purpose**: Goal or purpose of the project
- **Technologies Used**: Key tools, languages, frameworks
- **Results/Achievements**: What you accomplished with quantifiable metrics

### Example

**Website Optimization Project**
_Backend Developer, Fintech Solutions_
_Technologies Used: Python, PostgreSQL, Docker_
_Github URL: ..._

- Optimized database queries, reducing page load times by 30%.
- Implemented caching mechanisms that decreased server load by 15%.

## Tailor to the Job

Each project should reinforce your suitability for the role. If it doesn't align with job requirements, omit it or move it to a less prominent place.

## Where to Place Projects

- **Professional Experience**: If the project was part of your job, list under work experience
- **Education**: Include academic projects if you're a recent grad
- **Dedicated Projects Section**: If you have several impressive projects, create a "Key Projects" or "Selected Projects" section

## Rules

- ✅ List most relevant projects showcasing skills and expertise
- ✅ Brief description of each: purpose, role, technologies
- ✅ Focus on contributions and notable achievements
  - 👍 "Optimized database design to improve query performance, resulting in 20% reduction in page load times"
  - 👎 "Worked on the frontend, designing the user interface and implementing many features" (not an achievement)
- ❌ Don't include irrelevant or outdated projects
- ❌ Avoid listing incomplete or ongoing projects unless particularly impressive
